  1. < a >

  3. < address >

    4. Define la información de contacto del autor/propietario de un documento u artículo.

    Ejemplo:

  4. < article >

    5. Un artículo debe tener sentido por sí mismo y debe ser posible distriburilo independientemente del resto del sitio.

    Fuentes potenciales para el elemeto article:

    • Publicacion en el foro
    • Entrada en el blog
    • Noticia

    Nota: < article > no se muestra como algo especial en un navegador.Sin embargo, puede usar CSS para darle diseño

    Ejemplo article con css

  5. < aside >

    6. Define algún contenido además del contenido en el que se coloca. El contenido aparte de estar indirectamente relacionado con el conenido circulante.

    Sugerencia: aside se coloca a menudo como una barra lateral en un documento.

    Nota: Este elemento no se muestra como algo especial en el navegador. Sin embargo, puede usar CSS para diselar el aside

    Ejemplo aside con css

    7. Agrega controles de audio, como reprducción, pausa y volumen. El < source > permite especificar archivos de audio alternativos entre los que se puede elegir el navegador.

  7. < button >

    8. Define un boton en el que se puede hacer clic.

    Sugerencia: Especifique siempre el type de un < button >, para indicar a los navegadores qué tipo de botón es.

  8. < form >

    9. Es un contenedor para diferentes tipo de lementos de entrada, como : campos de texto, casillas de verificación, botones de opción, botones de envío, etc.

    • < input type="text">

      • Muestra un campo de entrada de texto de una sola linea


    • < input type="radio" >

      • Muestra un botón de radio(para seleccionar una de muchas opciones)

      Colores

    • < input type="checkbox" >

      • Nuestra una casilla de verificación (para seleccionar cero o más opciónes)

    • < input type="submit" >
    • < input type="button" >
    Ejemplo de form

  9. < footer >

    10. Define un pie de página para un documento o sección.

    Normalmente contienen:

    • Información de autoriía
    • Información de registro
    • Información de contacto
    • Mapa de sitio
    • Volver a los enlaces superiores
    • Documentos relacionados
    Ejemplo de footer

  10. < header >

    11. Representa un contenedor de contenido introductorio o un conunto de enlaces de navegación.

    Normalmente contienen:

    • Uno o más elementos de encabezado (< h1 > - < h6 >)
    • logotipo o icono
    • información de autoría

      • NOTA: Puede tener varios < header > en un documentos HTML. Sin enbargo, no se puede colocar dentro de un < footer >, < address > u otro < header >

  13. < img >

    14. Se utiliza para incrustar una imagen en una página HTML.

    Técnicamente, las imágenes no se insertan en una página web; las imágenes están vinculadas a páginas web. La etiqueta <ima> crea un espacio de espera para la imagen de referencia.

    Tiene tributos obligatorios:

    • src: especifica la ruta a la imagen
    • alt: especifica un texto alternativo para la imagen, si la imagen por algún motivo no se puede mostrar.">
